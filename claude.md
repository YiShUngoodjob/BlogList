1\. 引言

1.1 目的

本文档旨在明确“个人博客列表系统”的产品需求，包括功能特性、用户界面、交互流程及非功能性约束，为设计、开发及测试团队提供统一的工作依据。

1.2 范围

本项目为一个轻量级的个人博客展示平台，包含三个核心界面：

个人信息编辑页面（注册/登录后）

博客列表展示页面（大众浏览）

博客详情展示页面（个人完整信息）

系统以简约、白色为主色调，并融入磨砂玻璃（毛玻璃）视觉效果。

1.3 定义与缩写

术语 说明

磨砂玻璃效果 元素背景呈半透明模糊，类似毛玻璃质感，通常通过 backdrop-filter: blur() 实现

头像占位符 当用户未上传头像时，以姓名的第一个汉字作为替代显示

方框条目 列表中每个博客的展示卡片，圆角、半透明、带模糊背景

2\. 产品概述

2.1 背景与目标

在个人品牌展示日益重要的今天，许多用户希望拥有一个简洁的个人信息聚合页，便于社交分享或求职。本系统旨在提供一个极简的博客列表工具，让用户快速填写个人信息，生成专属的展示页面，并以美观的列表形式呈现给大众。

2.2 用户角色

注册用户：可填写/修改个人信息，生成个人博客。

访客：浏览博客列表，点击查看任意用户的详细信息。

2.3 产品特点

极简视觉设计，以白色为背景，减少视觉干扰。

独特的磨砂玻璃卡片设计，提升现代感。

聚焦个人核心信息展示，无多余社交功能。

3\. 功能需求

3.1 页面一：个人信息编辑页面

入口：用户完成注册/登录后自动进入，或从导航进入。

功能要求：

头像上传

支持点击上传图片（格式：JPG, PNG, GIF；大小 ≤ 5MB）。

上传后实时预览，若不上传则后续列表/详情页显示名字首字。

基本信息填写

姓名（必填，文本，20字符以内）

学校（可选，文本，50字符以内）

职业（可选，文本，30字符以内）

个人简介（可选，多行文本，200字符以内）

结构化信息填写

技能（可选，支持多标签输入，每个技能文本10字符以内，最多10个）

工作经历（可选，支持多段，每段包含：公司、职位、时间段、描述）

爱好（可选，多行文本或标签形式）

保存与预览

保存按钮：将数据提交至后端，成功后跳转到页面二（博客列表）。

支持实时保存草稿（可选需求）。

3.2 页面二：博客列表页面

入口：系统首页或所有用户公开访问的默认页。

功能要求：

列表展示

以卡片列表形式展示所有已创建博客的用户（按最近更新倒序）。

每个卡片（方框）包含：

头像区域：若有头像则显示头像（圆形裁切），否则显示姓名的第一个汉字（居中，大号字体）。

姓名（必显示）

职业/学校（可选，显示其一或精简显示）

卡片圆角：16px 或更大圆角。

磨砂玻璃效果：卡片背景为半透明白色（如 rgba(255,255,255,0.6)），背景模糊（如 backdrop-filter: blur(10px)）。

交互

点击任意卡片，跳转到对应的页面三（个人博客展示页面）。

悬停时卡片轻微放大或阴影变化，提升反馈。

空状态

若暂无用户博客，显示提示“暂无博客，快来创建第一个吧！”并引导注册。

3.3 页面三：个人博客展示页面

入口：点击列表卡片进入。

功能要求：

信息布局

顶部：头像（大尺寸，若无则显示名字首字大号占位）、姓名、学校、职业。

中部：个人简介卡片（磨砂玻璃效果）。

下部：分栏展示技能（标签形式）、工作经历（时间线样式）、爱好（文本）。

设计要点

所有信息块均采用磨砂玻璃卡片风格，保持视觉统一。

工作经历可折叠或展开（非必须）。

操作

提供返回按钮回到列表页。

若为本人查看（已登录），可提供“编辑”按钮跳转到页面一进行修改。

4\. 界面与交互需求

4.1 视觉风格

主色调：白色背景（#FFFFFF 或 #F5F5F5 微暖白），文字深灰色（#333333），辅助线浅灰。

磨砂玻璃卡片：

背景色：rgba(255, 255, 255, 0.5~0.7)

模糊度：blur(10px)

边框：可选极细白边框或阴影增加层次感。

圆角：卡片统一 16px，按钮 8px，头像圆形。

字体：无衬线字体，如 Inter, Roboto, 系统默认。

4.2 响应式设计

移动端适配：卡片宽度自适应，在小屏幕下方框列表变为单列，内边距适当缩小。

桌面端：列表可采用两列或三列布局（根据屏幕宽度）。

4.3 交互动效

卡片点击：轻微缩放的点击态效果。

页面切换：平滑过渡动画（如淡入淡出）。

5\. 非功能性需求

5.1 性能

页面加载时间：首屏 < 1.5s，图片懒加载。

列表页支持分页或无限滚动，每次加载 10 条数据。

5.2 安全性

头像上传需校验文件类型，防止恶意文件上传。

用户数据访问权限：未登录用户只能读取，不能编辑他人信息。

5.3 兼容性

支持主流浏览器最新版本（Chrome, Firefox, Safari, Edge）。

移动端适配 iOS 和 Android 浏览器。

5.4 可维护性

前端组件化开发，磨砂玻璃效果可抽象为通用卡片组件。

后端 API 遵循 RESTful 设计。
